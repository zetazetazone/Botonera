## Directory Purposes

**app/:**
- Purpose: Expo Router routing configuration and entry points
- Contains: TSX screen route definitions, layout providers
- Key files: `_layout.tsx` (root provider wrapper), `index.tsx` (home route)

**src/:**
- Purpose: Main application code organized by concern
- Contains: Components, screens, hooks, utilities
- Key files: All feature implementation

**src/components/:**
- Purpose: Reusable UI building blocks
- Contains: React Native presentational components
- Key files: `SoundButton.js` (grid item with 3-column responsive layout), `AddSoundFab.js` (FAB with WhatsApp green color), `EditSoundModal.js` (color picker + text input modal)

**src/screens/:**
- Purpose: Full-screen containers that coordinate components
- Contains: Logic for screen-level state, effects, event handling
- Key files: `HomeScreen.js` (sound grid + share intent + modal orchestration)

**src/hooks/:**
- Purpose: Custom React hooks for state logic and side effects
- Contains: Context providers, custom hooks
- Key files: `useSoundboard.js` (SoundboardContext provider + hook + AsyncStorage operations)

**src/utils/:**
- Purpose: Non-React utility functions for services and APIs
- Contains: Audio operations, file system bridges
- Key files: `audioBridge.js` (Expo Audio, FileSystem, Sharing, Haptics wrapper functions)

**hooks/:**
- Purpose: Expo template hooks (not actively used in application)
- Contains: Theme color hooks from Expo starter template
- Key files: Unused but present from initialization

**constants/:**
- Purpose: Application-wide constants and theme definitions
- Contains: Color palettes, typography
- Key files: `theme.ts` (Colors object with light/dark modes, Fonts platform-specific mappings)

**assets/:**
- Purpose: Static images and media
- Contains: App icon variants, splash screens, Android adaptive icons
- Key files: icon.png, splash-icon.png, android-icon-*.png

**android/:**
- Purpose: Native Android project files
- Contains: Gradle configuration, native source code
- Key files: Generated by Expo, not manually edited

**scripts/:**
- Purpose: Build and setup automation
- Contains: Node.js scripts for project maintenance
- Key files: `reset-project.js` (clears template artifacts)

**.vscode/:**
- Purpose: VSCode editor configuration
- Contains: Extension recommendations, workspace settings

**.planning/codebase/:**
- Purpose: GSD (Get Stuff Done) architecture and planning documents
- Contains: Analysis documents for codebase mapping
- Key files: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md

## Key File Locations

**Entry Points:**
- `app/_layout.tsx`: Root layout that wraps entire app with ShareIntentProvider and SoundboardProvider
- `app/index.tsx`: Home route definition that renders StatusBar + HomeScreen
- `src/screens/HomeScreen.js`: Main UI container with sound grid and share intent handling

**Configuration:**
- `app.json`: Expo configuration including plugins (router, splash-screen, share-intent), Android/iOS settings
- `package.json`: Dependencies (expo, react-native, @react-navigation, async-storage, expo-av, expo-file-system, etc.)
- `tsconfig.json`: TypeScript configuration extending expo/tsconfig.base with path alias @/* → ./
- `eslint.config.js`: ESLint configuration using expo lint preset

**Core Logic:**
- `src/hooks/useSoundboard.js`: Sound CRUD operations, AsyncStorage persistence, context provider
- `src/utils/audioBridge.js`: Audio playback, file system operations, sharing, haptics abstraction
- `src/screens/HomeScreen.js`: Sound grid rendering, share intent reception, modal state, playback management

**Components:**
- `src/components/SoundButton.js`: Responsive grid button with tap/long-press gestures
- `src/components/AddSoundFab.js`: Floating action button positioned bottom-right
- `src/components/EditSoundModal.js`: Modal with text input, color picker grid, save/delete/cancel buttons

**Theme & Styling:**
- `constants/theme.ts`: Color definitions for light/dark modes, platform-specific fonts

## Naming Conventions

**Files:**
- React components in src/components/: PascalCase.js (e.g., SoundButton.js, EditSoundModal.js)
- Screens in src/screens/: PascalCase.js (e.g., HomeScreen.js)
- Hooks in src/hooks/: camelCase.js starting with "use" (e.g., useSoundboard.js)
- Utilities in src/utils/: camelCase.js (e.g., audioBridge.js)
- Root routes in app/: camelCase.tsx or _layout.tsx for special files
- Config files: kebab-case or dot-notation (tsconfig.json, eslint.config.js)

**Directories:**
- Feature-based organization: `src/components/`, `src/screens/`, `src/hooks/`, `src/utils/`
- Expo template: `app/` for routing, `assets/` for static files
- Root level: lowercase single words (android, hooks, constants, scripts)

**Functions:**
- Async operations: camelCase with "Async" suffix or Async prefix (saveAudioFile, loadSounds, playSoundFile)
- Event handlers: handleX pattern (handleShortTap, handleLongPress, handleModalClose, handleSave, handleDelete)
- Getter functions: useX for hooks, getX or ensureX for utilities
- Component props: camelCase (onPress, onLongPress, onClose, onSave, onDelete, visible, sound)

**Constants:**
- Color arrays: UPPERCASE_SNAKE_CASE (COLORS, STORAGE_KEY, SOUNDS_DIRECTORY)
- Numeric constants: uppercase in components (numColumns = 3, padding = 16)

## Where to Add New Code

**New Feature (e.g., New Sound Collection):**
- Primary code: Create new screen in `src/screens/CollectionScreen.js`
- Add route: Define new route in `app/_layout.tsx` or add new file like `app/collection.tsx`
- State: Extend SoundboardContext in `src/hooks/useSoundboard.js` with new methods
- Components: Create UI components in `src/components/` if reusable
- Tests: Create `src/screens/CollectionScreen.test.js` (if testing exists)

**New Component/Module (e.g., Custom Color Picker):**
- Implementation: `src/components/ColorPicker.js` with export default function
- Props: Accept required data and callbacks (color, onColorChange, colors array, etc.)
- Styling: Use React Native StyleSheet.create() at bottom of file
- Usage: Import in parent component and pass props

**Utilities/Helpers (e.g., Sound Validation):**
- Shared helpers: `src/utils/soundValidator.js` with named exports (validateSoundTitle, validateAudioFile, etc.)
- Platform-specific: Use suffix pattern like `src/utils/storage.native.js` and `src/utils/storage.web.js` if needed
- Avoid: Don't create utilities that could be hooks if they use React APIs

**Global State (e.g., User Preferences):**
- Pattern: Create new Context file `src/hooks/usePreferences.js` with Provider component and hook export
- Location: Place alongside useSoundboard.js for consistency
- Integration: Wrap with provider in `app/_layout.tsx` alongside other providers
- Usage: Custom hook export (e.g., usePreferences) consumed via useContext

**Constants & Config (e.g., API Endpoints):**
- Application constants: `constants/config.ts`
- Theme colors: `constants/theme.ts`
- Magic strings/numbers: Declare at top of feature file or in constants/

## Special Directories

**android/:**
- Purpose: Native Android project
- Generated: Yes (created by Expo)
- Committed: Yes (version controlled for EAS builds)
- Modify: Only for native module integration, avoid manual edits

**.expo/:**
- Purpose: Expo CLI cache and configuration
- Generated: Yes
- Committed: No (in .gitignore)
- Modify: Not directly

**node_modules/:**
- Purpose: Dependency packages
- Generated: Yes (from package.json via npm install)
- Committed: No (in .gitignore)
- Modify: Never directly

**.planning/:**
- Purpose: GSD planning and architecture documentation
- Generated: Manually via /gsd commands
- Committed: Yes (part of design system)
- Modify: When codebase structure or patterns change
